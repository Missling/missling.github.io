Arrays and Hashes
January 9, 2015

Array is a list of elements that sit in between square brackets. Element includes, but not limited to strings, array, and integers. 

      countries = ["US", "Canada", "Mexico", "Costa Rica"]
      even_numbers = [ 2, 4, 10, 46, 100]
      random_array = ["cat", 456, [1, 2, 3], "blue", 6487589]

Each array element has a position identifier with the first element indexed at 0, and subsequent elements are indexed sequentially by an increase of 1. However, array index can start at the end of an array using the reversed index. The last element is indexed at -1, the second to last element is indexed at -2, and so forth. This might sound confusing, so I provided some examples below so you can have some visuals to follow along. 

      array = [5, 7, 8, 10]
      array[0] => 5
      array[-4] => 5 (using reversed index)
      array[0..2] => 5, 7, 8

Hash, on the otherhand, is a collection of key-value pairs. Keys within a hash must be unique, but the restriction does not apply to values. Any Ruby objects can be used as a key or value, so integers, strings and arrays can be applied to hashes as well. 

      Hash syntax:

      hash = {
        key1 => value1,
        key2 => value2,
        key3 => value3
      }
    
      Examples of hashes:

        animals = {
          "cats" => 1,
          "dog" => 2,
          "bird" => 1
        }

        food = {
          "japanese" => "sushi"
          "indian" => "curry"
          "italian" = "pasta"
        }

Unlike arrays, which use numbers for indexing, hashes can use arbitrary objects as their indexes. This means you can find the value relating to a specific key. 

      food["japanese"] => "sushi"
      food["italian"] => "pasta"

However, the same technique will not work for locating a specific key by using the value. 

      food["sushi"] does not give "japanese"
