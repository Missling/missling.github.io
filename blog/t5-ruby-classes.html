Ruby Classes
January 24, 2015

Today I will go over an important component of Ruby, classes. When I first started learning about classes, classes were explained to me as being factories within the Ruby language. Inside these factories, objects can be built and these objects are referred to as being ‘instances of the class’. Here we have a Car class, with attributes of make and model: 

    class Car
      def initialize(make, model)
        @make = make
        @model = model
      end
    end

Our goal is to make a new instance of the Car class. This is achieved by calling .new on our class and defining it’s make and model. .new actually calls the initialize method, which takes the make and model as it’s parameters and sets them as the attributes of your new car object. 

    car_1 = Car.new (“Honda”, “Civic”)

Within a class there can be methods that interact with the class object. Let’s create a method to that will paint our car object a specific color. 

    def car_color(color)
      “Spraying your car #{color} color.”
    end
To call our method on car_1: 
    p car_1.car_color(“blue”) #=> “Spraying your car blue color”

The method, car_color, is an instance method. As an instance method it can only be used after an instance of the class was created with the initialize method. From there, you can access the method by calling the method on the instance of the class that is created, in our case it would be car_1.

Going back to our initialize method, notice there are @ in front of the make and model. This means that make and model are actually instance variables, and instance variables are available to every method of the class. As instance variables, this is why make and model are accessible in the car_color method.

On the other hand, there are local variables. Local variables are only available inside the method it is defined, and not shared across the entire class. In our car_color method, color is a local variable, which means it can not be accessed outside the method. To turn color into an instance variable, I can do the following: 

    def car_color(color)
      @color = color
      “Spraying your car #{@color} color.”
    end

As @color, now it can be access outside of the car_color method.
