Ruby Hashes vs. JavaScript Objects
February 6, 2015

In Ruby, a collection of unique keys and their values is call a hash, whereas in Javascript, it is call an object. They might have a slightly different appearance and a different name, but under their exteriors they both work the same way.

The syntax of a Ruby hash: 

    hash = {
            Key 1 => value 1,
            Key 2 => value 2
            }

    animals = {
              “cat” => 1,
              “dog” => 2
              }
To add a new key-value pair to our animal hash: 
    animals[“bird”] = 1

    animals = {
              “cat” => 1,
              “dog” => 2,
              “bird” => 1
              }
To delete the bird key-value pair from our hash: 
    animals.delete(“bird”)
To access the value corresponding to a key: 
    animals[“cat”] will return 1
To modify a value: 
    animals["cat"] = 4

    animals = {
              “cat” => 4,
              “dog” => 2
              }
The syntax of a Javascript object: 
    object = {
              property 1: value 1,
              property 2: value 2,
              }

    var animals = {
                  cat: 1,
                  dog: 2
                  }
To add a new property-value pair to our animal object: 
    animals.bird = 1

    var animals = {
                  cat: 1,
                  dog: 2,
                  bird: 1
                  }
To delete the bird property-value pair from our object 
    delete animals.bird
To access the value corresponding to the cat property: 
    animals.cat will return 1
To modify the value corresponding to the cat property: 
    animals.cat = 4

    var animals = {
                  cat: 4,
                  dog: 2,
                  bird: 1
                  }
