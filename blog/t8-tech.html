Regular Expressions
February 11, 2015

Regular Expressions, also known as regexp, is use to determine whether a specific string contains a given pattern. Regexp is usually distinguishable by the forward slashes (/). In the example below, we will try to locate the pattern 'peanut' within the string 'organic peanut butter'. 

    /peanut/ =~ 'organic peanut butter'
    returns 8

By using =~, Ruby returns the index position of the first match that is found within the string. In the case of 'organic peanut butter', peanut starts at index position 8. If a match is not located, Ruby will return nil. 

    /peanut/.match('organic peanut butter')
    returns MatchData "peanut"

.match will return a MatchData followed by what was matched. In our case, Ruby returns the object, peanut. Similiar to =~, if no match is found, Ruby will return nil.

Ruby classifies the following as metacharacters: ( ) [ ] { } ? + * . In order to use metacharacters in regexp, back slashes are needed. 

    /peanut\+butter/.match('organic+peanut+butter')
    returns MatchData "peanut+butter"

Square bracket ([ ]) are used within regexp for character class. Character class tells regexp that it has to match with only one of the characters listed. For example, [ae] tells Ruby to look in the string for either 'a' OR 'e'. In contrast, /ae/ tells Ruby that in order for a match to occurred,'a' must exist followed by 'e'. 

    /[ae]/ =~ 'cat'
    returns 1

    /ae/ =~ 'cat'
    returns nil

In the example below, you can find a hyphen (-) in between the square brackets. The hyphen denotes a range, so [a-d] is the equivalent of [abcd]. 

    /[a-d]/ =~ 'robot'
    returns 2

    /[abcd]/ =~ 'robot'
    returns 2
