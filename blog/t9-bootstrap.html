Bootstrap

February 22, 2015

For this week challenge, we were asked to revamp our original website. Unlike the previous time, this time around we were given permission to use Bootstrap or Jekyll to assist us with the process. Since I have heard of Bootstrap from my developer friends, Bootstrap was the option that I went with.

In the beginning, I underestimated the amount of research I needed to do in order to use Bootstrap. I thought I could dive right in, with no research, and I was going to unleash some crazy, hidden Bootstrap talent. Instead, I realize I needed to spend a little time figuring out how to use Bootstrap. Once I figured out that I needed to include the Bootstrap files in my repository and how to navigate the Bootstrap website, the process to redo my website was pretty straight-forward.

While Bootstrap simplified the process to redo my website, the challenging part comes from the fact that Bootstrap gave me too many options. Before Bootstrap I took an exorbitant amount of time to simply create a button. To some degree Bootstrap turned the process from figuring out how to create a button to a shopping experience; which button should I choose? Although Bootstrap gave me more options, I still ran into the same issue when I created my website from scratch, deciding on the overall layout of my website. Unfortunately, Bootstrap didn't help in the creativity department.
