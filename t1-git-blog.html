Version Control, Git and Github

December 13, 2014

Version control is an essential tool that allows programmers the ability to save copies of their files at a particular point-in-time. In doing so, programmers can maintain a running revision history, granting them accessibility to past versions of their files. This optimizes the debugging process by allowing programmers a way to pin-point the exact version that the bug was introduced. Programmers can then revert back to working a previous, bug free version. To make the subject of version control more relatable to a non-programmer audience, version control can be applied to the context of video game. Prior to fighting the boss, players can save their current position in the game as a security measure. In the event that the player does die during battle, the player can continue playing from the last location they saved prior to the fight.

Git is a version control system that tracks changes by allowing users to save contents of their files locally, at it's current state. This is achieved through git add. Git add tells Git to turn modified files and new file that the users want to save into tracked files. From there, tracked files can be saved with git commit -m "message". By repeating the above steps, multiple versions of a file are saved, which creates a revision history showcasing the changes made to the code.

Github is important to the coding community because it provides a place where programmers can store their projects, Git repositories, remotely on the internet. Github's user-friendly platform allows programmers to easily view, compare and merge commits. The pull request feature on Github simplifies the merging process by allowing others to review, comment and approve code updates before being merged.
